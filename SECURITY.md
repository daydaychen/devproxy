## 安全注意事项

### 已知风险

**MITM 代理的本质**

devproxy 是一个 MITM（中间人）代理工具，这意味着它能够拦截和解密 HTTPS 流量。这是工具的核心功能，但也带来了安全责任：

1. **仅用于开发环境**: 本工具仅适用于开发和测试环境。请勿在生产环境中使用。
2. **证书安全**: 工具会生成自己的 CA 证书用于 MITM 拦截。这些证书仅应在受信任的环境中生成和使用。
3. **环境变量**: 工具会设置 `NODE_TLS_REJECT_UNAUTHORIZED=0`，这会禁用 Node.js 的证书验证。请确保只对目标子进程生效。

### 安全最佳实践

1. **不要提交敏感信息**: 确保配置文件中不包含真实的 API tokens、密码或其他敏感信息
2. **使用 .gitignore**: 敏感配置文件应被 git 忽略
3. **定期更新**: 保持工具和依赖库为最新版本
4. **最小权限**: 仅对需要代理的进程使用 devproxy，不要对整个系统范围使用
